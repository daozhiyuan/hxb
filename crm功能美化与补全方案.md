## 二、后端接口补全与权限

1. **客户列表** `/api/customers`  
   - 支持分页、搜索、筛选
   - 只返回当前合作伙伴的客户（管理员返回全部）

2. **客户详情** `/api/customers/[id]`  
   - GET：获取详情  
   - PATCH：编辑（合作伙伴只能编辑自己客户）

3. **新增客户** `/api/customers/register`  
   - POST：报备客户

4. **删除客户** `/api/customers/[id]`  
   - DELETE：仅管理员可用

5. **跟进记录** `/api/customers/[id]/follow-ups`  
   - GET/POST：获取/添加跟进

6. **客户导出** `/api/customers/export`  
   - GET：导出 Excel/CSV

7. **权限校验**  
   - 所有接口都需校验 session.user.role，删除接口需校验为 ADMIN

## 三、前端页面美化与功能完善

### 1. 客户列表页
- 使用 Radix UI Table + Tailwind 美化
- 搜索框、筛选、分页、导出按钮
- "新增客户"按钮（弹窗或跳转）
- "编辑"按钮（弹窗或跳转）
- "删除"按钮（仅管理员可见）
- "详情"按钮

### 2. 客户详情页
- 展示所有客户字段
- 跟进记录列表
- "添加跟进"按钮（弹窗）
- "编辑客户"按钮
- "删除客户"按钮（仅管理员）

### 3. 新增/编辑客户弹窗
- 使用 Radix Dialog + Tailwind 表单
- 字段校验、错误提示

### 4. 跟进记录弹窗
- Radix Dialog + Tailwind 表单
- 跟进内容、时间、类型

### 5. 通用美化
- 按钮、输入框、表格、弹窗、通知等全部用 Tailwind + Radix UI 组件
- 响应式布局，适配PC和移动端

## 四、自动化补全与美化实施方案

1. **后端接口**：自动补全所有接口文件，增加权限校验和字段校验。
2. **前端页面**：自动生成/美化客户列表、详情、编辑、跟进等页面，所有表单、弹窗、表格、按钮均用 Radix UI + Tailwind。

4. **导出功能**：提供导出按钮和后端接口，支持 Excel/CSV。
