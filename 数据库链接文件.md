# 数据库连接文件清单

本文档列出了项目中所有包含或使用数据库连接信息的文件，可用于配置和维护数据库连接。

## 核心数据库连接文件

### 1. Prisma配置文件

- **文件路径**: `/prisma/schema.prisma`
- **说明**: Prisma ORM的主要配置文件，定义了数据库连接方式和数据模型
- **连接配置**:
  ```prisma
  datasource db {
    provider     = "mysql"
    url          = env("DATABASE_URL")
    relationMode = "prisma"
  }
  ```

### 2. Prisma客户端实例

- **文件路径**: `/src/lib/prisma.ts`
- **说明**: 创建并导出Prisma客户端实例，用于在应用程序中访问数据库
- **连接配置**:
  ```typescript
  const databaseUrl = process.env.DATABASE_URL || "mysql://root:password@localhost:3306/crm_db";
  
  const prisma = global.prisma || new PrismaClient({
    datasources: {
      db: {
        url: databaseUrl,
      },
    },
    log: ['error', 'warn'],
  });
  ```

### 3. 数据库连接辅助模块

- **文件路径**: `/src/lib/db.ts`
- **说明**: 另一个Prisma客户端实例创建文件，包含更详细的连接配置
- **连接配置**:
  ```typescript
  return new PrismaClient({
    datasources: {
      db: {
        url: process.env.DATABASE_URL || "mysql://crmuser:crmpassword@localhost:3306/crm?connection_limit=5&pool_timeout=0&authentication_plugin=mysql_native_password"
      }
    },
    log: process.env.NODE_ENV === 'development' ? ['query', 'error', 'warn'] : ['error'],
  });
  ```

## Docker部署配置

### 1. Docker Compose配置

- **文件路径**: `/docker-compose.yml`
- **说明**: 定义了应用的容器化部署，包括MySQL数据库服务和应用服务
- **连接配置**:
  ```yaml
  # MySQL服务配置
  db:
    image: mysql:8.0
    environment:
      MYSQL_ROOT_PASSWORD: password
      MYSQL_DATABASE: nextn
    ports:
      - "3307:3306"
    # ...其他配置...
  
  # 应用服务配置
  app:
    # ...其他配置...
    environment:
      - DATABASE_URL=mysql://root:password@db:3306/nextn
      # ...其他环境变量...
  ```

### 2. 部署脚本

- **文件路径**: `/deploy.sh`
- **说明**: 自动化部署脚本，包含数据库初始化配置
- **数据库相关配置**:
  ```bash
  MYSQL_ROOT_PASSWORD=$DB_PASSWORD
  MYSQL_DATABASE=nextn
  # ...
  # 添加启动脚本到mysql-init目录
  cat > ./mysql-init/01-init.sql << 'EOF'
  ALTER DATABASE nextn CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
  # ...
  ```

## 环境变量

应用程序使用`.env`文件存储环境变量，包括数据库连接信息。主要的数据库相关环境变量有：

- **DATABASE_URL**: 数据库连接URL，格式为`mysql://user:password@host:port/database`
- **MYSQL_ROOT_PASSWORD**: MySQL根用户密码（Docker环境）
- **MYSQL_DATABASE**: 数据库名称（Docker环境）

## 注意事项

1. 项目同时存在多个数据库连接配置文件（`prisma.ts`和`db.ts`），应确保它们使用相同的连接信息
2. 部分脚本中硬编码了数据库连接信息，应考虑统一使用环境变量
3. 开发环境和生产环境的数据库连接配置可能不同，需要注意区分
