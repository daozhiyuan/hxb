# 客户管理系统部署指南

## 一、环境准备

### 1.1 系统要求
- 操作系统：Linux（推荐 Ubuntu 20.04+）
- Node.js：18.0.0+
- MySQL：8.0+
- 内存：最小 4GB，推荐 8GB+
- 存储：最小 50GB，推荐 100GB+

### 1.2 依赖安装
```bash
# 安装 Node.js
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt-get install -y nodejs

# 安装 MySQL
sudo apt-get install mysql-server

# 安装 PM2
npm install -g pm2
```

## 二、项目部署

### 2.1 获取代码
```bash
git clone [项目地址]
cd [项目目录]
```

### 2.2 安装依赖
```bash
npm install
```

### 2.3 环境配置
1. 复制环境变量模板：
```bash
cp .env.example .env
```

2. 编辑 `.env` 文件，配置以下关键变量：
```env
# 数据库配置
DATABASE_URL="mysql://user:password@localhost:3306/crm_db"

# 认证配置
NEXTAUTH_SECRET="your-secret-key"
NEXTAUTH_URL="http://your-domain.com"

# 加密配置（仅用于历史数据解密）
ENCRYPTION_KEY="your-legacy-key"

# 服务配置
PORT=3000
NODE_ENV=production
```

### 2.4 密钥管理
1. 生成 RSA 密钥对：
```bash
node scripts/generate-keys.cjs
```

2. 密钥文件说明：
- `private.pem`：私钥，用于解密数据
- `public.pem`：公钥，用于加密数据
- 请妥善保管私钥，建议使用 KMS 等密钥管理服务

### 2.5 数据库初始化
1. 创建数据库：
```sql
CREATE DATABASE crm_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

2. 执行数据库迁移：
```bash
npx prisma migrate deploy
```

### 2.6 数据修复与加密迁移
1. 备份数据库：
```bash
./backup.sh
```

2. 运行统一修复脚本：
```bash
node scripts/fix-encrypted-data.cjs
```

## 三、服务启动

### 3.1 开发环境
```bash
npm run dev
```

### 3.2 生产环境
1. 构建项目：
```bash
npm run build
```

2. 使用 PM2 启动服务：
```bash
pm2 start ecosystem.config.cjs
```

3. 查看服务状态：
```bash
pm2 status
pm2 logs
```

## 四、Nginx 配置

### 4.1 安装 Nginx
```bash
sudo apt-get install nginx
```

### 4.2 配置反向代理
```nginx
server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
```

### 4.3 启用 HTTPS（推荐）
```bash
sudo apt-get install certbot python3-certbot-nginx
sudo certbot --nginx -d your-domain.com
```

## 五、维护与监控

### 5.1 日志管理
- 应用日志：`logs/app.log`
- 错误日志：`logs/error.log`
- PM2 日志：`pm2 logs`

### 5.2 备份策略
1. 数据库备份：
```bash
# 每日备份
0 2 * * * /path/to/backup.sh
```

2. 密钥备份：
- 定期备份 `private.pem` 和 `public.pem`
- 建议使用加密存储或 KMS 服务

### 5.3 监控告警
1. 系统监控：
- CPU 使用率
- 内存使用率
- 磁盘空间
- 网络流量

2. 应用监控：
- 响应时间
- 错误率
- 并发连接数

## 六、故障处理

### 6.1 常见问题
1. 数据库连接失败
   - 检查数据库服务状态
   - 验证连接字符串
   - 确认网络连接

2. 加密/解密错误
   - 确认密钥文件存在
   - 检查密钥文件权限
   - 验证加密配置

3. 服务无法启动
   - 检查端口占用
   - 查看错误日志
   - 验证环境变量

### 6.2 应急处理
1. 服务降级：
```bash
pm2 stop all
pm2 start ecosystem.config.cjs --env production
```

2. 数据恢复：
```bash
./restore.sh [备份文件]
```

## 七、安全建议

### 7.1 系统安全
- 定期更新系统和依赖
- 使用防火墙限制访问
- 启用 SELinux 或 AppArmor

### 7.2 应用安全
- 启用 HTTPS
- 配置 CSP 策略
- 实现请求限流
- 启用 WAF

### 7.3 数据安全
- 使用 KMS 管理密钥
- 定期轮换密钥
- 加密敏感数据
- 实施访问控制
