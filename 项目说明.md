## 一、项目概述

这是一个基于 Next.js 开发的客户管理系统，主要用于管理客户信息、申诉处理及合作伙伴关系管理等功能。系统采用现代化的技术栈，提供友好的用户界面和完整的功能支持。

### 主要功能
- 客户管理（信息录入、查重、状态跟踪、列表查看）
- 申诉管理（申诉提交、状态跟踪、详情查看、列表管理）
- 文件管理（多格式上传、预览、安全存储）
- 用户角色权限（超级管理员、管理员、合作伙伴、普通用户）
- 跟进记录（会议、电话、邮件等类型跟进）

### 技术栈
- 前端：Next.js 14 + shadcn/ui + Tailwind CSS
- 后端：Node.js + Next.js API Routes
- 数据库：MySQL
- ORM：Prisma
- 认证：NextAuth.js
- 文件存储：本地文件系统

### 2. 路由调整记录
- 项目最初混用 App Router（`/src/app`）和 Pages Router（`/src/pages`），导致页面404。
- 解决方案：备份 `src/pages` 目录，统一采用 Next.js 13+ 的 App Router。
- 相关路由已全部迁移至 `/src/app` 目录下，确保系统路由一致性。

### 3. 新增目录和文件
- `/src/hooks/` - 自定义钩子函数，如移动设备检测(use-mobile)
- `/src/contexts/` - React上下文管理，包括UI和侧边栏状态
- `/src/providers/` - 全局状态提供者组件
- `/src/components/ui/` - 通用UI组件库，包含大量可复用的界面元素
- `/src/app/api/auth/session/` - 会话管理API，处理会话修复
- `/src/lib/decryptIdCard.ts` - 多格式加密ID号解密工具，支持Base64、IV:EncryptedHex格式和双重加密
- `/dist/decryptIdCard.js` - 编译后的解密工具，被修复脚本直接引用
- `/scripts/fix-all-double-encrypted-ids.js` - 用于扫描并修复数据库中双重加密或格式异常的ID号
- `/src/app/api/analytics/customers/route.ts` - 客户数据分析API，根据用户角色返回不同范围的客户分析数据
- `/src/app/api/analytics/users/route.ts` - 用户数据分析API，仅对管理员开放，提供用户分析数据
- `/src/app/crm/reports/page.tsx` - 报表页面，实现基于角色的权限控制
- `/src/app/profile/components/decryption-tool.tsx` - 超级管理员专用的证件号码解密工具组件
- `/src/app/api/admin/decrypt-idcard/route.ts` - 超级管理员专用的证件号码解密API，支持多种加密格式

## 三、数据模型设计

系统主要包含以下数据模型：

### 1. 用户模型 (User)
- 包含四种角色：超级管理员、管理员、合作伙伴、普通用户
- 字段：ID、姓名、邮箱、密码哈希、角色、状态、创建/更新时间

### 2. 客户模型 (Customer)
- 记录客户的基本信息和状态
- 字段：ID、姓名、公司名称、电话、邮箱、状态（跟进中、谈判中、待定、已签约、已完成、已流失）
- 高级信息：行业、优先级、来源、收入、地址、职位等
- 关联：注册合作伙伴、标签、跟进记录

### 3. 申诉模型 (Appeal)
- 记录客户申诉信息和处理状态
- 字段：ID、客户名称、身份证号码（加密存储）、申诉原因、证据、状态（待处理、处理中、已批准、已拒绝）
- 关联：提交合作伙伴、处理操作员、历史申诉记录、操作日志

### 4. 跟进记录模型 (FollowUp)
- 记录客户跟进过程和结果
- 字段：ID、内容、创建时间、类型（会议、电话等）、时长、地点、后续步骤等
- 关联：客户、创建用户

### 5. 客户标签模型 (CustomerTag)
- 用于客户分类和标记
- 字段：ID、名称、颜色、创建/更新时间

## 五、后端接口与权限

### 1. 客户管理接口
- **GET** `/api/customers`：获取客户列表，支持分页、搜索、筛选（合作伙伴仅看自己，管理员看全部）
- **GET** `/api/customers/[id]`：获取客户详情
- **POST** `/api/customers/register`：新增客户
- **PATCH** `/api/customers/[id]`：编辑客户（合作伙伴仅编辑自己客户）
- **DELETE** `/api/customers/[id]`：删除客户（仅管理员）
- **GET/POST** `/api/customers/[id]/follow-ups`：获取/添加跟进记录
- **GET** `/api/customers/export`：导出客户数据（Excel/CSV）
- **POST** `/api/customers/check-duplicate`：客户查重功能

### 2. 申诉管理接口
- **GET** `/api/appeals`：获取申诉列表
- **GET** `/api/appeals/[id]`：获取申诉详情
- **POST** `/api/appeals`：提交申诉
- **PATCH** `/api/appeals/[id]`：更新申诉状态（仅管理员）
- **GET** `/api/appeals/[id]/logs`：获取申诉操作日志

### 3. 用户管理接口
- **GET** `/api/admin/users`：获取用户列表（仅管理员）
- **POST** `/api/admin/users`：创建用户（仅管理员）
- **PATCH** `/api/admin/users/[id]`：更新用户信息（仅管理员）
- **DELETE** `/api/admin/users/[id]`：删除用户（仅超级管理员）
- **GET** `/api/admin/users/pending`：获取待审核用户列表（管理员和超级管理员）
- **GET** `/api/admin/partners`：获取合作伙伴列表（管理员和超级管理员）

### 4. 文件上传接口
- **POST** `/api/upload`：文件上传
- **GET** `/api/upload/[filename]`：文件下载/预览

### 5. 数据分析接口
- **GET** `/api/analytics/customers`：获取客户分析数据（所有用户，但普通用户仅能查看自己的客户数据）
- **GET** `/api/analytics/users`：获取用户分析数据（仅管理员和超级管理员）

### 6. 证件号码解密接口
- **POST** `/api/admin/decrypt-idcard`：专用于超级管理员解密各种格式的证件号码，包括URL安全的Base64格式

### 7. 权限控制
- 所有接口通过中间件校验 `session.user.role`
- 基于角色的访问控制 (RBAC)
- 资源所有权验证（合作伙伴只能访问自己的资源）
- 使用`isAdmin()`辅助函数统一验证管理员级别权限，支持ADMIN和SUPER_ADMIN角色
- 敏感数据解密API仅对超级管理员开放

## 九、安全特性

- NextAuth.js 认证与授权
- 敏感数据加密存储（如身份证号码）
  - 支持多种加密格式（标准IV:EncryptedHex、Base64、URL安全Base64）
  - 特殊格式加密数据支持（tkl1开头格式、短IV格式）
  - 多种解密尝试策略（不同IV长度、不同加密模式、多种偏移量）
  - 统一的解密方法，确保系统各模块（客户管理、申诉处理）一致性
  - 优化加密算法减少密文长度（减少IV长度，使用URL安全Base64）
  - 有效ID号码保持明文存储以节省数据库空间
  - 自动检测并修复双重加密问题
  - 专用解密API和工具，仅超级管理员可用
  - 多步骤加密和解密机制，确保数据安全
- 密码安全哈希存储
- 文件上传安全控制（类型限制、大小限制）
- API 访问权限控制
- 操作日志记录和审计
- SQL 注入防护（通过 Prisma ORM）
- XSS 防护（React 自带）
- CSRF 防护
